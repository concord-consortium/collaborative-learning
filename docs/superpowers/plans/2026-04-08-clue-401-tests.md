# CLUE-401 Test Suite Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add comprehensive tests for the click-to-pick-up tile feature (CLUE-401), covering UI state, keyboard navigation, drop zones, placement, cancel paths, model layer operations, and multi-pane behavior.

**Architecture:** Tests are organized by layer — model tests (pure MST, no DOM), component tests (React Testing Library + MobX providers). Each test file covers one unit with focused, independent tests. Follow existing patterns from `dc-tile-move-copy.test.ts`, `ui.test.ts`, and `delete-button.test.tsx`.

**Tech Stack:** Jest, React Testing Library, MobX State Tree, `specStores()`, `setupDocumentContent()`

### Task 5: Keyboard Navigation Tests

**Files:**
- Create: `src/components/document/pick-up-keyboard-nav.test.tsx`

This tests `handlePickUpKeyDown` behavior. Since it's a private method on `DocumentContentComponent`, test via dispatching `KeyboardEvent` on `document` while the component is rendered with a picked-up tile.

Render a `DocumentContentComponent` with stores that have `ui.pickedUpTileId` set and `ui.pickedUpDocId` matching the document's contentId. The component registers a `keydown` listener via a MobX reaction when a tile is picked up.

| Test | Setup | Assert |
|------|-------|--------|
| `ArrowDown focuses first drop zone` | pickUpTile, dispatch ArrowDown | `ui.focusedDropZoneIndex === 0` |
| `ArrowDown cycles forward` | focusedDropZoneIndex = 0, dispatch ArrowDown | `focusedDropZoneIndex === 1` |
| `ArrowDown stops at last zone` | focusedDropZoneIndex = lastIndex, dispatch ArrowDown | stays at lastIndex |
| `ArrowUp focuses last zone from unfocused` | pickUpTile, dispatch ArrowUp | `focusedDropZoneIndex === lastIndex` |
| `ArrowUp cycles backward` | focusedDropZoneIndex = 2, dispatch ArrowUp | `focusedDropZoneIndex === 1` |
| `ArrowUp stops at first zone` | focusedDropZoneIndex = 0, dispatch ArrowUp | stays at 0 |
| `Enter places tile at focused zone` | focusedDropZoneIndex = 1, dispatch Enter | tile moved, pickedUpTileId cleared |
| `Tab focuses delete button` | pickUpTile, dispatch Tab | delete button has focus (requires `.problem-panel` + `.delete-button` in DOM) |
| `Escape clears pick-up` | pickUpTile, dispatch Escape | `pickedUpTileId === undefined` |
| `skips processing when focus on interactive element` | focus on a `<button>`, dispatch ArrowDown | `focusedDropZoneIndex` unchanged |

**Note:** This is the hardest test group — rendering `DocumentContentComponent` requires substantial provider setup. If too complex, consider extracting `handlePickUpKeyDown` into a testable utility function that takes `(zones, ui, content)` and test that directly.
